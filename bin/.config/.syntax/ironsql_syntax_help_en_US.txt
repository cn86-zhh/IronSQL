[*]   Welcome to the IronSQL database syntax help documentation. We aim to design a personal,
[*] non-industrial-grade data management system. This will be an extremely long process.
[*] IronSQL's syntax design is very similar to MySQL, but compared to the massive MySQL system,
[*] IronSQL is lightweight. This also means that in a simple and easy-to-understand syntax
[*] environment, you don't need to spend too much time learning IronSQL syntax. Good designs
[*] always learn from each other.

[*] USE Syntax

[-] use <database_name>;
    |
    |__ Use the target database object for operations.
    |__ Note: When the target database object does not exist, this will cause the 'use target database object' to fail.

[*] CREATE Syntax

[-] create database <database_name>;
    |
    |__ Create a target database.
    |__ Note: When the target database already exists and you try to create the same database object, this will fail.

[-] create table <table_name>;
    |
    |__ Create a table object under the current database.
    |__ Note: When this table object already exists and you try to create the same table object, this will fail.

[*] DROP Syntax

[-] drop database <database_name>;
    |
    |__ Delete an existing target database object.
    |__ Note: When you try to delete a database object, you need to pay attention to the following points.
    |__ The database object must exist, the database object is not in use, the database object has no cross-database associations,
    |__ or the database association object has transferred ownership.

[-] drop table <table_name>;
    |
    |__ Delete an existing table object in the current database.
    |__ Note: When you try to delete a table object, you need to pay attention to the following points.
    |__ The table must exist, the table has no associations, or associations have transferred ownership,
    |__ the table has no cross-database associations, or the table's cross-database association objects have transferred ownership.

[*] SHOW Syntax

[-] show databases;
    |
    |__ Display all database objects.

[-] show tables;
    |
    |__ Display all table objects in the current database.

[-] show datas <table_name>;
    |
    |__ Select the specified table from the specified database to display table data.

[-] show struct from <database_name.table_name>;
    |
    |__ Absolute syntax:
    |__ Display the table database structure of the target table.

[-] show struct from <table_name>;
    |
    |__ Relative syntax:
    |__ Display the table data structure of the target table in the current database.

[-] show struct <table_name>;
    |
    |__ Select the specified table from the specified database to display table data structure.

[-] select * from <table_name>;
    |
    |__ Select the specified table from the specified database to display table data.
    |__ '*' matches all table data.
    |__ Example: select id,name,sex from user_info;
    |__ Explanation: In the current database, select the specified table column names to display table data.

[*] SELECT Syntax (Absolute Query)

[-] select * from <database_name.table_name>;
    |
    |__ Select the specified database to display the specified table data.
    |__ '*' matches all table data.
    |__ Example: select id,name,sex from users.user_info;
    |__ Explanation: Select the specified table column names from the specified database to display table data.

[-] show table struct from <database_name.table_name>;
    |
    |__ Select the specified database and table to display the specified table data structure.
    |__ Example: show table struct from users.user_info;

[*] LINK Syntax (Absolute Syntax, Relative Syntax)

[-] link table from <table1> to <table2> new <table3>;
    |
    |__ Relative syntax: Merge table data and generate a new table.
    |__ Detailed explanation: The target merge object only accepts two tables. The merged new table must not exist, otherwise the merge will fail.

[-] link table from <database1.table1> to <database1.table2> new <table3>;
    |
    |__ Absolute syntax: Merge table data and generate a new table.
    |__ Detailed explanation: The target merge object only accepts two tables. The merged new table must not exist, otherwise the merge will fail.

[-] link show table from <table1>,<table2>,...;
    |
    |__ Relative syntax: Link and display table data without creating a new table.
    |__ Detailed explanation: After using 'use <database_name>;', you can link and display table data. If the table does not exist, table data cannot be displayed.

[-] link show table from <database_name>:<table1>,<table2>,...;
    |
    |__ Absolute syntax: Link and display table data without creating a new table.
    |__ Detailed explanation: No need to use database object first, you can link and display multiple table data in the target database.

[-] link show table struct <table1>,<table2>,...;
    |
    |__ Relative syntax: Link and display the data structure of table data.

[-] link show table struct <database_name>:<table1>,<table2>,...;
    |
    |__ Absolute syntax: Specify the target database and link to display the table database of the target table.
